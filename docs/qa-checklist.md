# QA 체크리스트 (핵심 25개)

운영 전 최소 품질 보장을 위한 필수 체크리스트입니다.  
각 항목은 실제 화면 기준으로 확인하고, 이상이 있으면 재현 방법과 스크린샷을 남깁니다.

## 1) 로그인/회원가입

- [ ] 회원가입 성공 후 로그인 가능하다.
- [ ] 잘못된 비밀번호 입력 시 오류 메시지가 정상 노출된다.
- [ ] 로그아웃 후 보호 페이지(`/book`, `/my`, `/admin`) 접근 시 로그인으로 리다이렉트된다.
- [ ] 로그인 후 `redirect` 파라미터 경로로 정상 이동된다.

## 2) 예약/취소(회원)

- [ ] 잔여 횟수 0인 경우 예약 버튼이 비활성 또는 경고 노출된다.
- [ ] 잔여 횟수 1 이상이면 예약이 정상 생성된다.
- [ ] 예약 성공 시 잔여 횟수가 1 감소한다.
- [ ] 예약 취소 시 잔여 횟수가 1 증가한다.
- [ ] 예약/취소 정책 문구가 예약 화면 상단에 고정 노출된다.
- [ ] 예약 마감 시간(정책 시간) 이후에는 예약이 차단된다.
- [ ] 취소 마감 시간(정책 시간) 이후에는 취소가 차단된다.

## 3) 예약/취소(관리자)

- [ ] 관리자가 예약을 취소하면 회원의 잔여 횟수가 증가한다.
- [ ] 예약된 슬롯 삭제/비활성 시 회원 잔여 횟수가 증가한다.
- [ ] 예약된 슬롯 변경 시 경고/확인 메시지가 노출된다.

## 4) 관리자 대시보드/데이터 싱크

- [ ] 이번 주 슬롯 수가 DB와 일치한다.
- [ ] 이번 주 예약 수가 DB와 일치한다.
- [ ] 예약 현황(오늘/이번주) 총 예약 인원이 DB와 일치한다.
- [ ] 예약 현황에서 예약된 슬롯만 노출된다.
- [ ] 예약 현황에서 회원 정보(이름/연락처)가 정상 표시된다.

## 5) 출석/노쇼 처리

- [ ] 출석 버튼 클릭 시 상태가 “출석”으로 변경된다.
- [ ] 노쇼 버튼 클릭 시 상태가 “노쇼”로 변경된다.
- [ ] 상태 변경 후 목록이 즉시 반영된다.
- [ ] “미정” 상태는 UI에 표시되지 않는다(내부 상태만 유지).

## 6) 시간 슬롯 관리

- [ ] 주간 시간 편집에서 저장 시 슬롯이 DB에 반영된다.
- [ ] 저장 후 주간 시간 편집/슬롯 영역에 상태가 반영된다.
- [ ] 예약된 슬롯 삭제 시 경고/확인 후에만 진행된다.
- [ ] 이전 주/다음 주 복사 기능이 정상 동작한다.

## 7) 통계 탭

- [ ] 통계 탭이 관리자 대시보드에 노출된다.
- [ ] 주간/월간 전환이 정상 동작한다.
- [ ] 예약률/취소율/노쇼율/출석률이 표시된다.
- [ ] 피크 타임 및 회원별 이용 패턴이 노출된다.

## 8) 성능/보안 기본

- [ ] 로그인하지 않은 사용자가 `/admin` 접근 시 차단된다.
- [ ] 정적 자원(css/js)이 정상 로드된다(dev/prod 모두).
