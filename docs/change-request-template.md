# 기능 변경 요청 템플릿

아래 템플릿을 복사해서 요청하면, 구현/테스트/배포까지 빠르게 진행할 수 있습니다.

## 1) 기능 추가 템플릿

```md
목표: (무엇을 왜 추가하는지 1문장)
범위: (예: /admin > 회원 탭, /book 페이지)
요구사항:
1. 
2. 
3. 
예외조건:
1. 
2. 
데이터 영향: (DB 컬럼/테이블/RPC 변경 여부. 없으면 "없음")
완료기준:
1. 
2. 
테스트:
1. 수동 테스트 시나리오
2. 자동 테스트 추가 여부
브랜치/배포:
1. 작업 브랜치명
2. main 머지 여부
3. 푸시 여부
```

## 2) 기능 삭제 템플릿

```md
목표: (왜 제거하는지 1문장)
범위: (삭제할 화면/컴포넌트)
삭제 대상:
1. UI 요소
2. 로직/핸들러
3. API/DB 영향 여부
대체 UX: (삭제 후 사용자에게 보여줄 동작/문구)
완료기준:
1. UI에서 완전히 사라짐
2. 코드 참조 제거됨
3. 빌드/테스트 통과
브랜치/배포:
1. 작업 브랜치명
2. main 머지 여부
3. 푸시 여부
```

## 3) 빠른 요청 예시

```md
목표: 관리자 회원 탭에서 회원 메모를 추가/수정/삭제한다.
범위: /admin > 회원 탭
요구사항:
1. 회원별 메모 1개 저장
2. 저장 후 목록 즉시 반영
3. 공백 메모 저장 불가
예외조건:
1. 500자 초과 금지
2. 권한 없는 사용자는 접근 불가
데이터 영향: profiles.note 컬럼 사용
완료기준:
1. 관리자 계정에서 CRUD 동작
2. 빌드/테스트 통과
브랜치/배포:
1. codex/member-note
2. main 머지
3. 푸시
```
